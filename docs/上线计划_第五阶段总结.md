# 上线计划第五阶段总结：API 限流第一步

## 1. 文档目的

本文档用于沉淀 `docs/上线计划.md` 第五阶段当前已完成的第一小步，重点记录 API 入口限流的设计、实现边界与验收方式。

本次对应上线计划中的核心内容是：

- `Phase 5：并发治理与安全补强`
- `API 限流`

---

## 2. 本次为什么先做 API 限流

在第四阶段完成 Kafka、独立 worker、最小重试和 DLQ 后，系统已经具备了异步消费能力，但 API 入口仍然缺少最基本的流量保护。

如果这一层不先补上，当前系统仍然存在明显上线风险：

- 任意来源可以持续冲击 `POST /auth/login`
- 公共调试入口 `POST /chat/agent` 容易被直接打满
- 登录后的高耗时写接口可以被短时间重复提交
- API 线程和后续 Kafka / worker 链路会一起承压

相比“用户级并发控制”“会话级串行控制”这类需要进一步跨 API / worker / 共享存储协同的能力，API 限流更适合作为第五阶段第一步：

- 收益直接
- 改动面集中
- 不破坏现有接口结构
- 可以单独验收和单独回滚

---

## 3. 本次目标

本次目标是给高风险写接口增加最小入口保护，而不是一次性做完整流量治理平台。

完成标准如下：

- 高风险写接口命中限流时返回 `429`
- 不修改现有接口入参和成功响应结构
- 默认支持 Redis 共享计数，未配置 Redis 时本地仍可运行
- 健康检查和只读查询接口暂不误伤
- 测试可重复执行

---

## 4. 实际完成的内容

## 4.1 新增 API 限流服务

新增文件：

- `backend/app/services/rate_limit_service.py`

主要能力：

- 定义高风险接口限流规则
- 解析客户端标识（优先 `X-Forwarded-For`，其次 `request.client.host`）
- 默认优先使用 Redis 计数
- Redis 不可用时回退到 `InMemory`

## 4.2 接入全局中间件

调整文件：

- `backend/app/main.py`

当前行为：

- 请求进入 API 后先注入 `request_id`
- 若命中高风险限流规则，则直接返回 `429 Too Many Requests`
- 响应附带：
  - `Retry-After`
  - `X-RateLimit-Limit`
  - `X-RateLimit-Remaining`
- 请求日志中增加 `limited=true/false`

## 4.3 新增配置项

调整文件：

- `backend/app/core/config.py`
- `.env.example`

新增配置：

- `API_RATE_LIMIT_ENABLED`
- `API_RATE_LIMIT_WINDOW_SECONDS`
- `API_RATE_LIMIT_AUTH_MAX_REQUESTS`
- `API_RATE_LIMIT_CHAT_MAX_REQUESTS`
- `API_RATE_LIMIT_TASK_CREATE_MAX_REQUESTS`

## 4.4 当前纳入保护的接口

- `POST /auth/register`
- `POST /auth/login`
- `POST /auth/refresh`
- `POST /chat/agent`
- `POST /chat/agent/session`
- `POST /chat/agent/session/stream`
- `POST /chat/tasks`

---

## 5. 当前边界

这一步有意只做“入口限流”，暂未一次性覆盖以下能力：

- 用户级并发控制
- 会话级串行控制
- worker 侧并发数治理
- 登录失败次数锁定
- 审计日志
- 敏感日志脱敏

这些会继续留在第五阶段后续小步推进。

---

## 6. 测试与验证

本次新增的最小验证包括：

```bash
python -m compileall -q backend/app
cd backend
python -m pytest -q app/tests/test_api.py -k "rate_limit"
```

重点覆盖：

- 连续登录请求会在阈值后返回 `429`
- 连续调用公共聊天入口会在阈值后返回 `429`
- `GET /health` 不受本轮限流影响

---

## 7. 本次可验收结果

- 高风险写接口已具备最小入口保护
- 命中限流时有明确 HTTP 语义和响应头
- 现有接口结构未被破坏
- 本轮仍保持“小步可回滚”
